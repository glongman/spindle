"Tapestry is a powerful, all-Java framework for creating leading edge web applications in Java."  quoted from the  Tapestry Home Page.

Eclipse is a platform designed for building integrated development environments (IDEs) that can be used to create diverse applications. Except for a small run-time kernel, all of the Eclipse's functionality is located in plug-ins.

Spindle is one such plugin, built to bring excellent IDE support for creating and editing the various parts of web applications built using Tapestry.

Spindle is distributed as a single distribution, including source code.

The Spindle JAR was compiled on Windows 2000 using Sun's JDK 1.3 compiler.

This product includes software developed by the Apache Software Foundation ( http://www.apache.org/).

Building Spindle

Spindle is delivered to you ready to run, just skip down to  Installing Spindle and Getting Started .

Spindle is built using the the Eclipse platform, and distributables are built using the  Ant build tool, version 1.4 or better. You do not need to download and install Ant before installing and using Spindle.

Spindle can only be built within the Eclipse Platform itself. To obtain the entire Spindle project contents for Eclipse, the most efficent way is to download the project in its entirety via anonymous CVS.  

Instructions:

  1. One should review how Eclipse interacts with CVS before trying to import the Spindle Project.  Docs are easily found within Eclipse by browsing Help (Menu = Help->Help Contents).
  2. The anonymous CVS  information needed to fill out the New Repository Location Wizard can be found here: http://sourceforge.net/cvs/?group_id=50321
  3. Once connected to cvs.spindle.sourceforge.net, you can browse the HEAD and any versions. The HEAD contains the latest version of the spindle code (it may be broken at any particular time). I have had problems with CVS->Eclipse integration in the past so I don't have a version in CVS for every release.  No worries though as the CVS stuff in Eclipse 2.0 is rock solid.
  4. Select the version of spindle (com.iw.plugins.spindle) you wish to import, right-click and select "Check out as project". You now have the project but it does not build.
  5. To get Spindle to build:
  • To build the spindle project, you need to import  the plugins it depends on as Java Projects
  • Activate File->Import->External Plugins and fragments, a wizard will appear. Click "Next"
  • Select "Choose from plugins in the runtime workbench". Click Next
  • Select the following plugins:
    • org.eclipse.jdt.ui
    • org.eclipse.core.runtime
    • org.eclipse.jdt.core
    • org.eclipse.core.resources
    • org.eclipse.pde
    • org.eclipse.pde.ui
    • org.eclipse.pde.core
    • org.eclipse.update.ui.forms
  • Click "Add Required Plugins"
  • Click "Finish", the wizard will create projects for each selected plugin. The Spindle project build path is already set to look for these projects
  • Running Spindle, select "Run->Run As->Runtime WorkBench

Installing Spindle and Getting Started

This release of Spindle only supports the following release(s)/build(s) ( found here) of the Eclipse Platform:

Eclipse R2.0 (SDK)

Assuming that you have and installed a working Spindle compatible build of Eclipse (see above) into a directory, say C:\eclipse ...

Simply unjar the file  com.iw.plugins.spindle-1.0.3.jar into C:\eclipse\plugins and restart Eclipse. That's almost all there is to it!

Since Spindle is a plugin that adds IDE support to Tapestry, it only makes sense that you are building an app with Tapestry. So you need to at least have the required version of

This build requires Tapestry version 2.1
  • net.sf.tapestry.jar ; and
  • javax.servlet.jar (also found in the Tapestry distro)
in the build path of your project. The Spindle wizards depend on this at runtime.  

Invoking the Spindle wizards is as easy as selecting a package, right clicking, selecting New -> Other -> Spindle and the desired wizard.

Quick access the wizards is available by customizing an Eclipse perspective with wizard actions on the main toolbar. In the Eclipse menu bar, select Perspective -> Customize. Depending on the Eclipse build, you'll find Spindle Action Set from there. Select it and two Spindle wizard icons will appear in the main toolbar.

In

Eclipse R2.0 (SDK)

, the exact steps are: Window -> Customize Perspective -> Other -> Spindle Action Set .

Spindle sets its own editors as the default for files with the following extensions:   *.application , *.jwc, and *.html . Simply open any file with these extensions and the corresponding Spindle editor will open.

Migrating pre-Tapestry2.0.4 projects to Tapestry 2.1 


There are two issues to address when upgrading a Tapestry project from pre-Tapestry2.0.4 to 2.1:
  • The names of Tapestry packages changed from "com.primix.tapestry.*" to "net.sf.tapestry.*"
  • There is a new XML DTD (version 1.2) for .application and .jwc files. Tapestry remains backwards compatible with the old 1.1 version of the DTD. ie., it continues to read 1.1 files ok.
To use the latest and greatest from Tapestry you need to migrate your existing projects. Note though that old versions of Tapestry are  not forward compatible with the package renaming or DTD 1.2 . If you migrate and use an older version of Tapestry at runtime, your app will croak.

To accomplish migration Spindle 1.0.3 provides a facility that will upgrade your .application and .jwc files to the new DTD.

As well it will scan your .application and .jwc looking for strings starting with "com.primix.tapestry" and changes them to start with "net.sf.tapestry".

So "com.primix.tapestry.BaseComponent" becomes "net.sf.tapestry.BaseComponent".

Note that common strings like EngineClass (Field Binding values also) are changed but not all stings are examined.
You may have to tweak your app a bit, but much less than you would otherwise.

Steps to migrate:
  1. First ensure that the name of the Tapestry jar file in your project build path starts with "net.sf.tapestry" and not "com.primix.tapestry". If  it does not, replace the jar and fix your build path before proceeding.
  2. Next make sure the Spindle Action Set described in Installing Spindle and Getting Started is activated, you can tell if it is if you see spindle icons on the main toolbar.
  3. To migrate the package imports of your .java files, Multi-Select all the packages in your project. unfortunately you can't just select the project (or the source folder(s)).
  4. hit CTRL-SHIFT-O or Select from the main menu bar... Source->Organize Imports. Eclipse will fix the java imports in all the packages you selected.
  5. Close all open editors and select the project you wish to migrate Tapestry files in. The Spindle migration function will be disabled unless the project (and only the project) is selected.
  6. To migrate the .application and .jwc files, click on the "M" icon in the Main Toolbar. It will perform the migration. You should notice that all the .application and jwc files in the PackageExplorer now have (DTD 1.2) appended to thier name.
  7. That's it. BTW its not a problem if one migrates tapestry files more than once by accident - the Migrator will ignore files that do not require migration.

Known Issues

Note that in spindle-distro-1.0.0-EarlyAccess-TAP-2.0.4-ECLIPSE-F3 there is a bug that wrote out incorrect DTD information.

This was the cause of [ 574242 ] Spindle is looking for the Tapestry DTD
.

It is easy to note in Spindle 1.0.3 which files have this problem, they show up in the PackageExplorer with (DTD ?) appended to thier name.

The easiest way to fix this is to migrate as in Migrating pre-Tapestry2.0.4 projects to Tapestry 2.1.

You can also fix the problem by hand. to do this open the .application or .jwc file in question. Look at the source page and ensure the DTD version is 1.2.

To fix, just change the "Howard Ship" to "Howard Lewis Ship"
