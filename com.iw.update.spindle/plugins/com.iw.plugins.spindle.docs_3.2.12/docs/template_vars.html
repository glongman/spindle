Template Variables

There are global variables that users can add to any template:

 Variable   Description 
 ${cursor}   Specifies the cursor position when the template edit mode is left. This is useful when the cursor should jump to another place than to the end of the template on leaving template edit mode. 
 ${date}   Evaluates to the current date. 
 ${dollar}   Evaluates to the dollar symbol '$'. Alternatively, two dollars can be used: '$$'. 
 ${file}   Evaluates to the name of the file. 
 ${time}   Evaluates to the current time. 
 ${user}   Evaluates to the user name. 

In addition, there are variables defined for each of the New File templates that Spindle wizards will fill in automatically. It is not possible for the user to change how the values are determined(Although, user can edit the value afterwards!).

 Variable   Description   Context(where var is valid) 
 ${encoding}   always UTF-8 for now   Application, Library, Component, Page 
 ${publicId}   the public id of the Tapestry DTD shipped with Spindle (right now its the Tapestry 3.0 DTD)   Application, Library, Component, Page 
 ${publicIdUrl}   the URL of the Tapestry DTD shipped with Spindle   Application, Library, Component, Page 
 ${description}   Wizards do not resolve a value so 'description' will be the replacement   Application, Library, Component, Page 
 ${applicationName}   The 'name' attribute of the application tag (wizards will insert the project name   Application 
 ${engineClass}   The 'engine-class' attribute of the application tag (wizards will insert org.apache.tapestry.engine.BaseEngine  Application 
 ${homePage}   Wizards will insert the path of the application home page   Application 
 ${componentClass}   the 'class' attribute of the component-specification tag (wizard will insert the fully qualified name of the class chosen by the user)   Component 
 ${allowBody}   the 'allow-body' attribute of the component-specification tag (always replaced with 'yes'   Component 
 ${allowInformal   the 'allow-informal-parameters' attribute of the component-specification tag (always replaced with 'yes')   Component 
 ${pageClass}   the 'class' attribute of the page-specification tags (wizard will insert the fully qualified name of the class chosen by the user)   Page 

Users are free to use, or omit, these variables in thier own templates. Note that if you omit a variable, you must ensure that you supply equivalent markup to that which would be supplied when Spindle resolves that variable.

Also, note the context that each variable is valid in. If you use a variable in a context where it is not valid, the variable will not be resolved.
