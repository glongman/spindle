Web Context Locator

Identifier:
com.iw.plugins.spindle.core.webContextLocator

Since:
Spindle 3.1.19

Description:

Plugins can contribute implemtations of IProjectMetaLocator keyed on a nature id. Thus, projects that have a nature can provide web project meta information.

Spindle will defer writing its own metadata file (.tapestryplugin) if it finds a contribution that supplies the information it needs. This is to avoid duplication of information and to avoid the possiblity of metadatum files getting out of sync.

The first contribution that supplies a valid result wins.

Note: that using this extension point may result in your plugin being loaded unexpectedly. Spindle will defer instantiating locators until it absolutely needs to but when it does, contributing plugins will be loaded.

Configuration Markup:

<!ELEMENT extension (metadataLocator+)>

<!ATTLIST extension

point CDATA #REQUIRED

id    CDATA #IMPLIED

name  CDATA #IMPLIED>


<!ELEMENT metadataLocator EMPTY>

<!ATTLIST metadataLocator

natureId CDATA #REQUIRED

locator  CDATA #REQUIRED>

  • natureId - The id of the contribution defining a project nature.
  • locator - a class that implements com.iw.plugins.spindle.core.IProjectMetadataLocator
