"Tapestry is a powerful, all-Java framework for creating leading edge web applications in Java."  quoted from the  Tapestry Home Page.

Eclipse is a platform designed for building integrated development environments (IDEs) that can be used to create diverse applications. Except for a small run-time kernel, all of the Eclipse's functionality is located in plug-ins.

Spindle is one such plugin, built to bring excellent IDE support for creating and editing the various parts of web applications built using Tapestry.

Spindle is distributed as a single distribution, including source code.

The Spindle JAR was compiled on Windows 2000 using Sun's JDK 1.3 compiler.

This product includes software developed by the Apache Software Foundation ( http://www.apache.org/).

What's New

  1. ported to Eclipse F3
  2. early support for Tapestry 2.0.4

    NOTE: This early access version of Spindle reads Tapestry files using the 1.1 and 1.2 version of the Tapestry DTD.

    HOWEVER it only writes files using the 1.2 DTD

    Files written to the 1.2 DTD can not be read at runtime by Tapestry versions pre-2.0.4

    Eventually, this branch of the code will be folded into the root when the Tapestry API enables Spindle to detect and write either DTD version at the users whim 

    You have been warned.

  3. The New Component and New Page Wizards give the option to the user to add the new item to an existing application.
  4. The template used by the optional auto-generated html file in the New Component and New Page Wizards can be changed:
    • open any HTML file in the Spindle HTML Editor
    • change the file to look like the template you want the wizards to use.
    • right click anywhere in the file.
    • select the option to replace the template with the contents of the editor.
    • you can revert to the default template by choosing that option in the same pop-up menu.

Building Spindle

Spindle is delivered to you ready to run, just skip down to  Installing Spindle and Getting Started.

Spindle is built using the the Eclipse platform, and distributables are built using the  Ant build tool, version 1.4 or better. You do not need to download and install Ant before installing and using Spindle.

Currently, It is possible to build Spindle using the sources provided in this release within the Eclipse Platform only . That's not likely to change as Eclipse provides compile-time libraries for plugins.

Instructions on how to build plugins are available here.

Installing Spindle and Getting Started

This release of Spindle only supports the following release(s)/build(s) ( found here) of the Eclipse Platform:

Stable build F3

Spindle probably will not work work in Integration or nightly builds in the Eclipse 2.0 development stream other than those listed above.

A version of Spindle that works with Stable build F3 and Tapestry 2.0.2a  is available here

A version of Spindle that works with Stable build F1 and F2 is available here

A version of Spindle that works with Stable build 20020214 is available here

A version of Spindle that works with Stable build 20020321is available here

All the old versions of Spindle require differing versions of Tapestry - check thier respective README files.

Assuming that you have and installed a working Spindle compatible build of Eclipse (see above) into a directory, say C:\eclipse ...

Simply unjar the file  com.iw.plugins.spindle-1.0.0-EarlyAccess-TAP-2.0.4-ECLIPSE-F3 .jar into C:\eclipse\plugins and restart Eclipse. That's almost all there is to it!

Since Spindle is a plugin that adds IDE support to Tapestry, it only makes sense that you are building an app with Tapestry. So you need to at least have the required version of

This build requires Tapestry version 2.0.4
  • net.sf.tapestry.jar ; and
  • javax.servlet.jar (also found in the Tapestry distro)
in the build path of your project. The Spindle wizards depend on this at runtime.  

Invoking the Spindle wizards is as easy as selecting a package, right clicking, selecting New -> Other -> Spindle and the desired wizard.

Quick access the wizards is available by customizing an Eclipse perspective with wizard actions on the main toolbar. In the Eclipse menu bar, select Perspective -> Customize. Depending on the Eclipse build, you'll find Spindle Action Set from there. Select it and two Spindle wizard icons will appear in the main toolbar.

In build F3, the exact steps are: Window -> Customize Perspective -> Other -> Spindle Action Set .

Spindle sets its own editors as the default for files with the following extensions:  *.application , *.jwc, and *.html . Simply open any file with these extensions and the corresponding Spindle editor will open.
