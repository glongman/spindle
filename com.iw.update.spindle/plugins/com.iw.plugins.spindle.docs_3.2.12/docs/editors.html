Spindle Editors

Spindle provides two text editors (actually XML editors) for working on Tapestry artifacts:

  • A SpecificationEditor (Spec Editor) for .application, .library, .page, and .jwc files; and
  • A Template Editor for working with Tapestry template files.

A general list of common editor features (more specific features are described on the respective pages for each editor):

 Feature  Desc  Spec Editor  Template Editor 
 Syntax Coloring  yes  yes - jwcid's are made more prominent 
 XML Content Completion  Tapestry or W3C DTD based  yes  yes - if enabled* 
 Tapestry Completions  for attribute values  yes  yes - in jwcid's and component parameter attibutes only 
 "As you type revalidation"  in well formed and located documents**  yes  yes*** 
 XML outline  see the structure of the markup in the editor outline  yes - can switch to Tapestry outline  no 
 Tapestry based outline  An outline that shows the Tapestry 'bits' of the file  yes - can switch to the XML outline  yes 
 Integration with the "Link To" option of the Package Explorer  yes  yes 

* The template editor can offer content XHTML 1.0 completion proposals (strict, transitional, or frameset. You choose which on the Spindle preference pages).

** While revalidation will always catch malformed XML or HTML, Tapestry specific validations can only occur if the document is well formed and can be seen by the spindle Project Builder.

*** Revalidation in templates uses the Tapestry template parser. That parser stops at the first error encountered. It may appear that the "as you type" parser has missed an error. It hasn't.

Content completion proposals may appear to be ordered funny. The general rule is if a proposals are sorted according to this setup:

  1. The proposal is a required Tapestry specific (i.e. a required component parameter)
  2. The proposal is an optional Tapestry specific one(i.e. an optional component parameter)
  3. The proposal is for a required XML tag or attribute name.
  4. The proposal is for an optional XML tag or attribute name.

icons next proposals are colored brighter (reds or purples) for required ones an less so (greens or blues) for optional ones.

Spindle editor keybindings

 Keybinding  Description  Notes  Spec Editor  Template Editor 
 (Ctrl-O)  XML based Quick Search  find things fast in large documents  yes  no 
 (Ctrl-F3)  Tapestry based Quick Search  a more compact search that doesn't show all the XML markup  yes  no 
 (Ctrl-Shift-F)  Markup Formatting  yes*  no 
 (Alt+F6)  Jump to template  Important notes about jumping in general**  yes  N/A 
 (Alt+F7)  Jump to spec  N/A  yes 
 (Alt+F8)  Jump to Java class  yes  yes 
 (Ctrl+Alt+Arrow Keys  Navigate between XML attributes  yes  yes 

* Formatting is broken and thus disabled in Spindle 3.0 + Eclipse 3.0. This has been addressed and formatting works fine in Spindle 3.1

** Jumping (Alt+F6 and Alt+F7) are also available in the Java Editor. Note that jumping from a binary file or binary class to another binary file (binary means located in a jar file) is not possible. This is not a limitation of Spindle. Rather it is a limitation of Eclipse. Lastly, jumping depends on the project having been built without fatal errors and the current file being edited has been seen by the Spindle project builder.
