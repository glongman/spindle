Pseudo Variables

You can put in your own variables and when the template is inserted (by content assist) you can tab over to it and enter a value. There is no facility to resolve values for your custom variables automatically.

It might be tempting to name your own variable with a string you intend to be the default value. Something like...

<property-specification class="${com.mypack.MyObject}"/>

Variable names must be valid according to the rules for Java identifiers. That means, unfortunately, that the above template is invalid....

The Eclipse template parser will reject the variable because '.' characters are not allowed.
